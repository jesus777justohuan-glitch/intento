# Desafio_fibonacci_primo


Un espacio interactivo diseñado para el desarrollo y simulación de proyectos lógicos, aplicando conceptos clave como la secuencia de Fibonacci y las propiedades de los números primos a soluciones del mundo real.



## 📁 Estructura del Proyecto

La organización de los archivos y carpetas en este repositorio se distribuye de la siguiente manera:

```text
├── index.html                           # Página de inicio y bienvenida principal
├── css/
│   ├── index.css                        # Estilos y reglas responsivas del Inicio
│   ├── fibonacci.css                    # Diseño y componentes del reto Fibonacci
│   └── primos.css                       # Estilos y cajas de la sección Primos
├── html/
│   ├── fibonacci_ahorro_progresivo.html # Interfaz del Simulador de Ahorro
│   └── primos.html                      # Interfaz del Cifrador con Números Primos
├── img/
│   └── fibonacci-spiral-amp-golden-ratio-vector-21098716.avif
└── javascrip/
    ├── fibonacci.js                     # Lógica del cálculo de ahorro y viabilidad
    └── primo.js                         # Algoritmo de verificación y cifrado XOR
```

    
## 🚀 Características Clave

Simulador Fibonacci (Ahorro Progresivo): Permite proyectar un plan de ahorro basado en la secuencia matemática y evaluar la viabilidad financiera real de la curva exponencial en un mes.

Guardianes de la Web (Cifrado con Primos): Demuestra el uso de la asimetría matemática mediante un sistema interactivo que verifica números primos y cifra contraseñas utilizando operaciones lógicas complejas bit a bit (XOR).

Diseño Unificado y Responsivo: Interfaz oscura elegante (predominio de gris plomo con acentos en café avellana) calibrada de manera simétrica para computadoras y pantallas móviles.


## 🛠️ Tecnologías Utilizadas

HTML5 Semántico: Estructuración limpia de formularios, bloques lógicos y contenedores.

CSS3 Personalizado: Uso avanzado de variables globales (:root), Flexbox, selectores específicos y bloques @media para un comportamiento responsivo fluido.

JavaScript (Vanilla): Procesamiento de algoritmos matemáticos en tiempo real y manipulación dinámica de elementos de la interfaz mediante document.getElementById().

### Explicación de los puntos más esenciales de la estructura:

* **El archivo `index.html` en la raíz:** Es fundamental que esté afuera de todas las carpetas. GitHub Pages (o cualquier servidor de hosting) busca por defecto este archivo en la raíz para saber cuál es la portada que debe renderizar primero.
* **El árbol visual (Bloque `text`):** El uso de los caracteres gráficos (`├──`, `│`, `└──`) sirve para que cualquier programador que entre a tu repositorio entienda en menos de 5 segundos dónde está el código de estilos, dónde están las vistas adicionales y dónde reside la inteligencia del sistema. 
* **La carpeta `javascrip/`:** Se especifica tal cual la nombraste en tu sistema para evitar errores de ruta comunes cuando otras personas intenten clonar tu proyecto en Git.
* **Sección de Características y Tecnologías:** GitHub valora mucho los repos
